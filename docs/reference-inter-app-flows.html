Inter-App Data Flows

How data moves between T&Q, Segmentation, Capacity, and Sales Forecasting

Reference Field Guide

All four RPM apps are designed to work together as a suite. This page is a field reference for the data flows between them — what moves, how it moves, when to run it, and what breaks if you get the sequence wrong.

⚠ Not Automatic Today

None of these connections are automatic. Every data flow described here requires a deliberate manual export/import step or a configured ADO pipeline. Native app-to-app connections are on the roadmap. Until then, plan for these steps explicitly in your project timelines — they are not checkbox items.

The Four Apps and What They Exchange

🎯
Account Segmentation
Classifies every account into a segment
Segments → T&Q
Counts → Capacity
🗺
Territory & Quota
Designs territories and calculates rep quotas
Quotas → Forecasting
📊
Capacity Planning
Calculates headcount needs by role and geo

Flow 1: Segmentation → T&Q

ItemDetail
What movesAccount segment attribute per account (e.g. Strategic, Named, Commercial, SMB)
Why it mattersT&Q uses the segment attribute in the Rule Engine and in the Addressable Market calculation for target allocation
How it moves todayExport account-level segment data from Segmentation → format as a flat file → import into T&Q Data Hub as an account attribute
When to run itAfter finalizing segment definitions, before territory carving in T&Q. Re-run whenever segmentation is re-executed.
What breaks if skippedRule Engine rules that reference segment attributes will have no data to match against. Territory carving based on segment mix will be inaccurate.
Future stateNative ADO pipeline — Segmentation output view pushes directly to T&Q account import. Not yet available.

Flow 2: Segmentation → Capacity

ItemDetail
What movesAccount counts per segment per geography (e.g. 1,200 Strategic accounts in US-West)
Why it mattersCapacity coverage ratios (1 Strategic AE per 5 accounts) are applied against segment account counts to calculate suggested headcount. Wrong counts → wrong headcount recommendations.
How it moves todayExport segment count summary from Segmentation → import into Capacity coverage module as the account universe by segment
When to run itAfter finalizing segmentation, before running the Capacity tops-down suggested headcount calculation. Re-run whenever segmentation changes materially.
What breaks if skippedCapacity coverage ratio calculations use stale or zero account counts. Suggested headcount numbers will be meaningless.
Impact of re-segmentationMoving 500 accounts from Commercial to SMB directly changes how many Commercial AEs and SMB AEs Capacity says you need.

Flow 3: Capacity → T&Q

ItemDetail
What movesTerritories-per-headcount (TPH) recommendations — how many territories of each type are needed per geography
Why it mattersT&Q territory hierarchy design should be informed by how many reps Capacity says you need. If Capacity says US-West needs 12 Commercial AEs, design 12 Commercial territories — not 8, not 16.
How it moves todayManual reference — planners look at Capacity bottoms-up hiring plan output and use those numbers to inform T&Q hierarchy structure. No data export required.
When to run itBefore finalizing the T&Q territory hierarchy. Inform territory design, then lock the hierarchy.
What breaks if skippedTerritory structure is designed before headcount is modeled. Common result: too many or too few territories per region, requiring a hierarchy redesign after the headcount model runs.

Flow 4: T&Q → Sales Forecasting

ItemDetail
What movesRep-level quota by quota type by period (output of the Calc Quota by Quota Types module in T&Q)
Why it mattersSales Forecasting tracks forecast attainment against quota. Without quota data, attainment metrics have no denominator — the app shows pipeline data with no context.
How it moves todayExport quota data from T&Q (Calc Quota by Quota Types module) → import into Sales Forecasting quota module
When to run itAfter quotas are finalized and approved in T&Q, before Sales Forecasting goes live with users. Re-run whenever quotas are revised mid-year.
What breaks if skippedSales Forecasting shows pipeline data but no attainment tracking. Reps and managers see forecast calls but can't compare them to quota. The primary value of the app is not available.
PrerequisiteT&Q must be live and quotas approved before this flow makes sense. Sales Forecasting is almost always a second engagement for this reason.

Recommended Execution Sequence

When implementing multiple apps simultaneously, follow this sequence to avoid rework:

1
Configure and generate Segmentation — classify the account universe. Validate segment distribution with the customer before moving on.
2
Export segment counts → load into Capacity — coverage ratios now have accurate account data to work with.
3
Run Capacity headcount model — confirm TPH output by role and geography.
4
Design T&Q territory hierarchy informed by Capacity output — the right number of territories per region per segment type.
5
Export segment data → load into T&Q — account segment attribute available for Rule Engine and Addressable Market.
6
Complete T&Q configuration and generate — hierarchy is right, segment data is loaded, Rule Engine has attributes to work with.
7
Finalize and approve quotas in T&Q — once quotas are live and approved, the Sales Forecasting integration becomes meaningful.
8
Export quota data → load into Sales Forecasting — attainment tracking is now live. Roll out to users.
💡 In Practice

Most customers don't implement all four apps simultaneously. T&Q usually goes live first, Segmentation and Capacity are added in a second phase, and Sales Forecasting follows once T&Q quotas are trusted. Don't force the ideal sequence on a customer who isn't ready for it. Start where the pain is highest — usually T&Q — and expand from there.
