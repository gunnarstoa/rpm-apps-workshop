T&Q Module Reference

T&Q requires five core data objects, with a sixth (Product) almost always required in practice. Understanding their structure before configuration sessions prevents the most common data-related delays. The Data Prep Workbook (available in Resources) contains templates for each object.

Territory File Schema

FieldTypeNotes
Territory IDText (unique)Primary key. Use a stable code (e.g. "S01") not a display name
Territory NameTextDisplay name (e.g. "Northeast")
Territory ParentText (Territory ID)ID of the parent territory - defines the hierarchy rollup
Territory ManagerText (User ID)Optional - the manager responsible for this territory
⚠ V2 vs V3 Territory Format

V2: One flat file with all nodes appearing as rows, each row showing the node and its respective parent node. V3 (ADO): One separate source view per hierarchy tier. If you’re configuring V3, you need a separate file per tier — not one combined flat file.
