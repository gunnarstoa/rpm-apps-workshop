Lab: Segmentation & Capacity Configurator Exercise

Using the CloudWorks case study — Segmentation and Capacity sections

Exercise Overview

Using the same CloudWorks case study from Day 1, configure the Segmentation and Capacity sections of the configurator. The case study has a dedicated section for each — scroll down past the T&Q section to find them.

Access

  • Use the same configurator app you used in the T&Q exercise
  • Find the Segmentation section (lower in the same nav)
  • Then the Capacity section below that
  • Reference dataset files are the same — same accounts, employees, and opportunities from Day 1
After each configurator page, hit Apply / Calculate before moving on — same rule as T&Q. Don't skip this step or your downstream pages may not reflect your inputs correctly.

Part 1: Segmentation Exercise

1 Data Setup

  • Review the CloudWorks account data for firmographic attributes: industry, employee count, revenue bands
  • Identify which attributes should be list-formatted for use in segmentation rules — remember, numeric values can't be used directly as rule dimensions
  • Configure appropriate attributes with Enable for Calculate where applicable
  • Check that employee count and revenue are represented as banded attributes (not raw numbers)

2 Rule-Based Segmentation

CloudWorks uses 4 segments: Strategic, Named, Commercial, SMB.

  • Configure the segment list in the configurator with these four segments
  • Identify which account attributes will serve as rule dimensions:
    • Industry
    • Employee band
    • Revenue band
    • Wallet size tier (if configured)
  • Enable these attributes as rule dimensions

3 Score-Based Segmentation

  • Configure scoring metrics from the case study data
  • Assign weights — must sum to 100%
  • Define score-to-segment mapping thresholds (e.g., 7–10 = Strategic, 4–7 = Named/Commercial, 0–4 = SMB)

4 Wallet Size

CloudWorks uses the Revenue method for wallet size calculation.

  • Navigate to the Wallet Size configuration page
  • Select Revenue method
  • Configure the revenue-based wallet size page with appropriate addressable market % assumptions from the case study
Don't overthink the wallet size exercise — use simple revenue-based assumptions for the exercise. The point is to understand the configuration flow, not to tune the model precisely.

Part 2: Capacity Exercise

1 Role Hierarchy

CloudWorks has the following role structure:

  • Job Family Groups: Core, Support
  • Job Families: AE, CS, Pre-Sales, Management
  • Sales Roles: Strategic AE, Commercial AE, SMB AE, Customer Success Manager, Pre-Sales Engineer, Regional Manager
  • Configure 3 levels: Job Family Group → Job Family → Sales Role
  • Do NOT enable role grade for this exercise — it's not needed for CloudWorks and adds unnecessary complexity

2 Coverage Assumptions

From the CloudWorks case study, configure these coverage ratios at the Americas level:

Role Segment Coverage Coverage Ratio
Strategic AE Strategic + Named 1 AE : 5 accounts
Commercial AE Commercial 1 AE : 30 accounts
SMB AE SMB 1 AE : 100 accounts

Map overlay roles as derived coverage:

  • 1 generalist per 8 AEs
  • 1 manager per 10 AEs

3 Revenue Target

  • CloudWorks receives revenue targets at the sub-region level, by account segment
  • Configure accordingly in the Revenue Target page
  • Note: this dimensionality (sub-region + segment) will require post-gen routing work — be aware of this before generating

4 Tops-Down Setup

  • Configure tops-down at the Americas level
  • Configure 2 scenarios (base + conservative)

5 Bottoms-Up Setup

  • Configure bottoms-up entry at the territory level
  • Configure approval at the region level
If you're doing both Segmentation and Capacity, the data setup is shared — you only need to configure accounts, employees, and opportunities once. Don't re-enter the same data in the Capacity section if you've already done it for Segmentation.
