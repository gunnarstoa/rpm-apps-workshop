Limitations Quick Reference

What each RPM app can and cannot do today — as of March 2026

Reference March 2026
⚠ Use This Honestly

This page documents real limitations from the March 2026 training session. Use it to set accurate expectations with customers before they ask. A limitation surfaced proactively builds trust; the same limitation discovered mid-project damages it.

Suite-Wide Limitations

Capability Area What It CAN Do What It CANNOT Do Today Workaround
App-to-App Connections Data connection points exist in the app design (e.g., Segmentation output table in T&Q) No automatic data sync between apps. Finalized Segmentation data does not automatically flow into T&Q. Manual export/import or ADO pipeline between apps. Plan for this as an integration step in the project.
CoModeler Access CoModeler can read model structure, generate line item dictionaries, explain formula logic Not self-serve for customers or partners. Access is gated — requires the Apps team to enable it. Coordinate with the Apps team (Chan/Jon) to enable CoModeler for a specific customer workspace before demos or labs.
Configurator Access Configurator produces a fully configured, generated model Not self-serve for customers. Configurator access is gated to the Apps team currently. Chan and Jon handle generation for the first 3 partner projects. After that, partners run independently with Apps team oversight.
ADO Data Flows Inbound data from source systems (Salesforce, Snowflake, HRIS) into Anaplan via ADO No bidirectional ADO currently. Data flows one way: source systems → Anaplan. No write-back. None yet. Bidirectional ADO (Anaplan → Salesforce, Snowflake, Databricks) is on the roadmap.
ADO Streaming Scheduled batch data refresh from source systems Not real-time. Batch only — data has a refresh lag. Set expectations with customers that pipeline data will not be instant. Agree on a refresh cadence.

Territory & Quota (T&Q)

Capability Area What It CAN Do What It CANNOT Do Today Workaround
Boomerang Employees Handle employee LOA and return using effective-dated assignments at monthly granularity V3 assignment model uses monthly granularity only. Cannot track day-level precision for employees who leave and return within the same month. Extension required for day-level precision. First, challenge whether the customer truly needs day-level vs. month-level — most don't.
Territory Format (V3 Migration) V3 uses an improved intersection model (territory × employee × time) V3 territory format is a breaking change from V2. Existing V2 configurations and data structures are not directly compatible with V3. Plan a V2 → V3 migration project. Treat it like a faster re-implementation, not an upgrade. Well-documented V2 extensions make this significantly faster.
In-Year Hierarchy Changes Supports effective-dated territory and resource assignments to handle mid-year changes Changing the dimensionality of the hierarchy (number of tiers, planning measures) after generation requires a configurator re-run. This resets post-gen work. Get hierarchy decisions right before generating. Document the configurator settings and do not change dimensionality post-gen without planning for a re-run.
Optimizer — Territory Exclusion Preserve existing account assignments; exclude specific accounts from optimizer consideration Cannot exclude a specific territory from being optimized against (e.g., a placeholder "bucket" territory). Known gap. Track in the backlog. Advise customers to run optimization before populating placeholder territories.
Split Allocation (V2) V2 T&Q supports basic split account management via extensions Split Allocation is not natively supported in V2 base app. Requires an extension. Native in V3 (May 2026). If customer needs this in V2, scope it as an extension. Consider waiting for V3 if timeline allows.
Quota Acknowledgement Two options: (1) In-app seller view of their quota. (2) DocuSign connector for digital signature. DocuSign integration is an extension — not included out of the box. Scope as a common extension. Well-understood effort. High value for customers with formal comp acknowledgement requirements.

Account Segmentation

Capability Area What It CAN Do What It CANNOT Do Today Workaround
T&Q Integration A Segmentation output table exists in the T&Q model for receiving segment data No automatic sync. Finalizing the planning hierarchy in Segmentation does not push data to T&Q automatically. Manual export/import or ADO pipeline. Plan this as an explicit integration step in the delivery.
UCP Exclusion Logic Supports complex exclusion rules by firmographic criteria; account-level override in both directions The "exclude rule" checkbox UI is counterintuitive (checked = rule is active = accounts are excluded). Easy to misconfigure. Test exclusion rules carefully before committing. Brief customers on the double-negative checkbox behavior before they use the UX.
Score Threshold Changes Score-to-segment thresholds are configurable post-generation without a re-run Changing scoring metric weights after generation requires understanding where in the module those weights live. Not obvious to new users. Use CoModeler (when available) to locate the weight line items. Document the threshold settings during delivery.
Wallet Size Data Wallet size calculation is built into the app and supports multiple calculation methods Requires third-party firmographic data (e.g., employee count, industry benchmarks) that many customers don't have. Customers can use estimates or industry benchmarks. Alternatively, skip wallet size setup if it won't be used in scoring or target allocation.

Capacity Planning

Capability Area What It CAN Do What It CANNOT Do Today Workaround
In-Year Hierarchy Changes Supports effective-dated hiring plan entries; scenario comparison Changing planning hierarchy dimensionality (geo tiers, role structure) post-generation requires a configurator re-run. Get dimensionality decisions right before generating. The configurator conversation is more important than the post-gen work.
Segmentation Auto-Sync Coverage ratios can incorporate segment counts Segment count data from the Segmentation app does not automatically feed into Capacity coverage ratio inputs. Manual export/import or ADO pipeline. Small ADO setup effort once ADO is configured for the customer.
OWP Integration Can be connected to Anaplan Workforce Planning (OWP) for position and cost data Not automatic — requires a dedicated integration project. Only relevant if the customer has OWP live. Scope as a separate extension if customer has OWP. High value for customers with mature headcount planning processes.
Scenario Count Supports multiple planning scenarios (base, conservative, optimistic, etc.) Number of scenarios is set at generation time in the configurator. Adding scenarios post-gen requires extending the scenario list and replicating logic. Ask customers how many scenarios they need before generating. Default to 3 unless there's a specific reason for more.

Sales Forecasting

Capability Area What It CAN Do What It CANNOT Do Today Workaround
Salesforce Write-Back Read opportunity data from Salesforce via A4S or ADO; display it in Anaplan Cannot write forecast decisions back to Salesforce. Data flows inbound only. None currently. Bidirectional ADO is on the roadmap. Set this expectation clearly before go-live — do not promise write-back.
Real-Time Pipeline Updates Batch pipeline data ingestion on a scheduled basis Not real-time. Sellers won't see pipeline changes reflected instantly. Agree on a refresh schedule (e.g., nightly). Brief sales teams that the tool is a planning and forecasting layer, not a live CRM view.
Configurability Works well out of the box for structured forecast collection and scenario modeling No configurator — significantly less customizable than T&Q, Segmentation, or Capacity. Resist the urge to extend heavily before the customer has used the base app for one quarter. Let real usage identify where extensions are genuinely needed.
ICM Connection Tracks forecast attainment against T&Q quotas No native ICM connection — incentive compensation calculations are not available in the current app. ICM app is on the roadmap (TBD 2026). If customers need ICM now, this is a gap. Do not imply it will be available before you confirm with the Apps team.
